# Prompt 06: Documentar el Skill con el template

## Objetivo

Usa este prompt cuando el Skill ya tenga suficiente definición y quieras documentarlo formalmente.

Este paso debe usar obligatoriamente `template-documentar_skill.md`.

---

## Prompt principal

```text
Quiero documentar formalmente este Skill usando el template del proyecto.

Información disponible:

[pega aquí el diseño del Skill, evaluación, notas o resumen]

Usa obligatoriamente `template-documentar_skill.md`.

Completa la documentación formal del Skill en Markdown.

Sigue las reglas de documentación del proyecto.

No inventes datos críticos. Marca `Pendiente`, `Supuesto` o `Por validar` cuando corresponda.

Al final, agrega una sección breve llamada “Pendientes antes de validar el Skill”.
```

---

## Prompt para revisar la documentación final

```text
Revisa esta documentación formal de Skill:

[pega aquí la documentación]

Evalúa si está lista para usarse.

Entrega:

1. Diagnóstico.
2. Problemas encontrados.
3. Recomendaciones.
4. Versión corregida si hay información suficiente.
5. Pendientes finales.
```

---

## Resultado esperado

Al terminar este paso deberías tener:

- Documentación formal del Skill.
- Pendientes identificados.
- Checklist de validación.
- Base lista para probar o ajustar.
