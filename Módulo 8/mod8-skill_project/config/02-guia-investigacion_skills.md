# Guía de investigación sobre Skills de ChatGPT

## Propósito

Esta guía se usa cuando el usuario necesita entender, investigar, explicar o enseñar qué son los Skills de ChatGPT.

Debe ayudar a construir una base conceptual clara, práctica y verificable.

---

## Objetivo de esta guía

Ayudar a explicar:

- Qué son los Skills.
- Para qué sirven.
- Cómo funcionan conceptualmente.
- Cuándo tiene sentido usarlos.
- En qué se diferencian de prompts, proyectos o GPTs personalizados.
- Qué riesgos, límites o confusiones pueden existir.
- Cómo aterrizar el concepto en casos reales.

---

## Enfoque recomendado

Cuando se solicite investigar o explicar un tema relacionado con Skills, usa esta estructura:

1. Definición del tema.
2. Explicación en lenguaje simple.
3. Aplicación práctica.
4. Ejemplo concreto.
5. Riesgos o errores comunes.
6. Checklist de validación.
7. Recomendaciones para implementación.

Adapta la profundidad al tipo de solicitud:

- Para principiantes, prioriza claridad, analogías simples y ejemplos.
- Para diseño de Skills, explica solo los conceptos necesarios para tomar decisiones.
- Para comparación entre herramientas, usa criterios prácticos y evita respuestas absolutas.
- Para capacitación, incluye actividades, preguntas guía o ejemplos aplicados.

---

## Capa conceptual

Usa esta capa cuando el usuario pregunte:

- Qué es un Skill.
- Cómo funciona.
- Para qué sirve.
- Cuándo se usa.
- Qué lo diferencia de otras funciones.

La respuesta debe explicar el concepto sin asumir conocimiento técnico previo.

---

## Capa comparativa

Cuando el usuario pregunte por diferencias entre herramientas, compara usando criterios como:

| Elemento | Uso recomendado | Cuándo no conviene | Señal práctica |
|---|---|---|---|
| Prompt | Para tareas simples o puntuales | Cuando se requiere consistencia recurrente | La instrucción cabe en una conversación |
| Proyecto | Para mantener contexto, fuentes e investigación acumulada | Cuando se necesita una capacidad reutilizable específica | Hay varios archivos, referencias o conversaciones conectadas |
| GPT personalizado | Para una experiencia conversacional especializada | Cuando solo se necesita una tarea concreta y modular | Se necesita un asistente con identidad, tono y propósito estable |
| Skill | Para flujos repetibles, estructurados y reutilizables | Cuando la tarea cambia demasiado o es única | Hay una capacidad específica que debe activarse en contextos similares |

## Ejemplos de Skills que pueden investigarse

Incluye ejemplos aplicados a negocio, UX, investigación y transformación digital:

- Skill para resumir entrevistas de usuarios.
- Skill para convertir hallazgos UX en insights accionables.
- Skill para auditar interfaces digitales.
- Skill para generar reportes ejecutivos.
- Skill para analizar feedback de clientes.
- Skill para transformar documentos extensos en briefs.
- Skill para crear checklists de investigación.
- Skill para evaluar madurez digital.
- Skill para documentar procesos internos.
- Skill para convertir datos en recomendaciones de negocio.
- Skill para estructurar workshops de co-creación.
- Skill para revisar entregables UX/UI.
- Skill para convertir investigaciones en oportunidades de producto.

---

## Reglas de investigación

Cuando investigues sobre Skills:

- Diferencia claramente entre conceptos confirmados, inferencias y recomendaciones.
- No inventes capacidades técnicas.
- Si un dato puede haber cambiado, indica que debe verificarse con documentación oficial actualizada.
- Prioriza fuentes oficiales cuando se hable de capacidades de ChatGPT, OpenAI o Skills.
- Usa ejemplos prácticos para aterrizar conceptos.
- Señala limitaciones, supuestos y riesgos.
- Distingue entre una recomendación metodológica y una capacidad real de la plataforma.
- Evita presentar una metodología propia como si fuera documentación oficial.
- Cuando el usuario necesite tomar una decisión, termina con una recomendación aplicable.

Las respuestas basadas en esta guía deben ser:

- Claras.

- Aplicadas.
