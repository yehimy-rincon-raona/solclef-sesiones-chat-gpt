# Investigación sobre Skills de ChatGPT

Este proyecto tiene como objetivo investigar, documentar y estructurar criterios para comprender qué son los **Skills de ChatGPT**, cuándo conviene crearlos, cómo evaluarlos y cómo aplicarlos en contextos de negocio, UX, investigación y transformación digital.

---

## Criterios para identificar si algo debería convertirse en Skill

Cuando el usuario describa una tarea, se debe analizar si conviene convertirla en un Skill usando los siguientes criterios.

---

### Alta probabilidad de convertirse en Skill

Una tarea tiene alta probabilidad de convertirse en Skill cuando:

- Se repite con frecuencia.
- Tiene pasos claros.
- Requiere consistencia.
- Usa una plantilla o formato fijo.
- Necesita seguir reglas específicas.
- Se beneficia de ejemplos o archivos de referencia.
- Puede automatizar parte del trabajo.
- Tiene entradas y salidas relativamente estables.

---

### Baja probabilidad de convertirse en Skill

Una tarea tiene baja probabilidad de convertirse en Skill cuando:

- Es única o muy puntual.
- Cambia demasiado cada vez.
- No tiene un flujo claro.
- Depende principalmente de criterio creativo abierto.
- No requiere consistencia.
- Puede resolverse con una simple instrucción de conversación.

---

## Matriz de evaluación de oportunidades para Skills

Cuando se quieran evaluar ideas de Skills, se puede usar la siguiente matriz:

| Criterio | Pregunta guía | Puntaje 1-5 |
|---|---|---|
| Frecuencia | ¿La tarea se repite muchas veces? | |
| Claridad | ¿El proceso tiene pasos claros? | |
| Valor | ¿Ahorra tiempo o mejora calidad? | |
| Consistencia | ¿Necesita resultados estandarizados? | |
| Reutilización | ¿Sirve para más de un caso? | |
| Complejidad | ¿Tiene suficiente complejidad para justificar un Skill? | |
| Recursos | ¿Necesita plantillas, archivos o scripts? | |

---

### Interpretación del puntaje

| Puntaje total | Interpretación |
|---|---|
| 25-35 puntos | Muy buen candidato para Skill. |
| 15-24 puntos | Puede funcionar como Skill si se define mejor. |
| Menos de 15 puntos | Probablemente basta con una instrucción o plantilla. |

## Reglas de investigación

Cuando se investigue sobre Skills, se deben seguir estas reglas:

- Diferenciar claramente entre conceptos confirmados, inferencias y recomendaciones.
- No inventar capacidades técnicas.
- Si un dato puede haber cambiado, indicar que debe verificarse con documentación actualizada.
- Priorizar fuentes oficiales cuando se hable de capacidades de ChatGPT, OpenAI o Skills.
- Usar ejemplos prácticos para aterrizar conceptos.
- Señalar limitaciones, supuestos y riesgos.

Las respuestas deben entregarse en un formato claro, estructurado y reutilizable.

- Marcos de análisis.

## Preguntas iniciales sugeridas para el proyecto

Cuando se inicie una nueva investigación o diseño de Skill, se pueden usar las siguientes preguntas:

1. ¿Qué proceso quieres convertir en Skill?
2. ¿Quién lo usaría?
3. ¿Qué problema resuelve?
4. ¿Qué entrada recibiría?
5. ¿Qué salida debería entregar?
6. ¿Qué reglas debe seguir siempre?
7. ¿Qué ejemplos reales tienes de uso?
8. ¿Qué archivos, plantillas o referencias debería consultar?
9. ¿Qué errores debe evitar?
10. ¿Cómo sabremos que el Skill funciona bien?
