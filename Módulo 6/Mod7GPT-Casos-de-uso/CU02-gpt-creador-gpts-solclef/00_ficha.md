# CU02 - Ficha del caso de uso - GPT Creador de GPTs Solclef

> Módulo 07 - GPTs personalizados - Solclef  
> Documento formativo ficticio para practica de creacion de GPTs

---

## Nombre del caso de uso

**Diseno guiado de GPTs internos Solclef mediante prompt maestro**

Caso de uso orientado a crear un GPT personalizado a partir de una necesidad de negocio, usando el ayudante creador de GPTs y un prompt maestro como punto de partida.

---

## Escenario

Los equipos de una organizacion de energia renovable pueden identificar muchas oportunidades para usar GPTs personalizados: resumir documentos, preparar reportes, crear formatos, analizar informacion, estructurar reuniones, revisar comunicaciones o apoyar tareas recurrentes.

Sin embargo, convertir una idea inicial en un GPT bien configurado puede ser dificil si no se define con claridad el objetivo, el usuario, las entradas, los limites y la salida esperada.

Este caso de uso propone usar un prompt maestro para guiar al ayudante creador de GPTs y obtener una primera configuracion estructurada.

---

## Problema que resuelve

| Situacion actual | Impacto |
|------------------|---------|
| Las ideas de GPT suelen comenzar de forma amplia o ambigua | Dificultad para convertirlas en una configuracion concreta |
| El usuario no siempre sabe que instrucciones debe escribir | GPTs con comportamiento poco consistente |
| Se crean GPTs sin definir limites claros | Riesgo de respuestas fuera de alcance |
| No siempre se piensa en entradas y salidas esperadas | Resultados dificiles de validar |
| Cada persona configura el GPT de forma distinta | Menor consistencia entre casos de uso |

El prompt maestro no sustituye el criterio de diseno. Ayuda a ordenar la conversacion con el creador de GPTs y a generar una primera version que luego debe revisarse y ajustarse.

---

## Objetivo del caso de uso

Construir un GPT personalizado usando el ayudante creador de GPTs a partir de un prompt maestro.

El participante partira de una necesidad sencilla, por ejemplo: "crear un GPT que ayude a preparar minutas de seguimiento", y usara el prompt maestro para obtener una propuesta de nombre, descripcion, instrucciones, iniciadores y prueba inicial.

El resultado esperado es un GPT configurado de forma clara, con alcance definido y criterios basicos de validacion.

---

## Alcance del GPT creado en el ejercicio

### El GPT debe hacer

- Ayudar a convertir una necesidad inicial en un GPT mejor definido.
- Formular preguntas de aclaracion cuando falte informacion critica.
- Proponer nombre, descripcion, instrucciones e iniciadores de conversacion.
- Definir entradas esperadas, reglas de trabajo, limites y formato de salida.
- Recomendar capacidades del GPT segun el caso de uso.
- Proponer una prueba inicial para validar el comportamiento.
- Mantener un lenguaje claro, profesional y orientado a negocio.

### El GPT no debe hacer

- Crear GPTs para reemplazar decisiones tecnicas, legales, financieras o laborales criticas.
- Inventar politicas internas, datos confidenciales o procesos no entregados.
- Proponer integraciones, acciones o APIs si no se explican sus condiciones de uso.
- Saltar directamente a instrucciones finales si la idea aun es ambigua.
- Presentar una configuracion como definitiva sin validacion humana.
- Usar informacion externa como fuente principal del diseno.

## Usuarios previstos

| Usuario | Uso esperado |
|---------|--------------|
| Lideres de area | Convertir necesidades operativas en ideas de GPT |
| Equipos de transformacion digital | Estructurar casos de uso internos |

---

## Valor esperado

- Acelerar la creacion inicial de GPTs personalizados.
- Mejorar la claridad del objetivo y alcance de cada GPT.
- Reducir configuraciones improvisadas o incompletas.
- Facilitar la comparacion entre diferentes ideas de GPT.
- Promover una practica responsable de diseno: entradas, limites, rutas y salidas.

---

## Herramienta utilizada

GPT personalizado con:

- Prompt maestro para conversar con el ayudante creador de GPTs.
- Archivo de ideas de GPTs cargado como insumo de practica.
- Instrucciones generadas y revisadas durante el ejercicio.
- Canvas activado para editar o refinar la configuracion.
- Analisis de datos activado si se requiere estructurar tablas o matrices.
- Busqueda web desactivada.
- Generacion de imagenes desactivada.
- Acciones sin configurar.

---

## Resultado esperado del ejercicio

Al finalizar, el participante debe tener:

1. Una idea de GPT seleccionada desde el archivo `insumos/ideas-gpts-solclef.md` o definida por el grupo.
2. Un prompt maestro completado con la necesidad del caso.
3. Una propuesta inicial generada por el ayudante creador de GPTs.
4. Un GPT creado con nombre, descripcion, instrucciones e iniciadores.
5. Una prueba inicial ejecutada en la vista previa del GPT.
6. Una breve nota con los ajustes realizados despues de revisar la primera propuesta.

---

## Criterios de exito

| Criterio | Verificacion |
|----------|--------------|
| La necesidad inicial se convierte en una configuracion clara | Si / No |
| El GPT tiene objetivo, usuarios, entradas, limites y salida esperada | Si / No |
| Las instrucciones no son genericas ni excesivamente amplias | Si / No |
| Los iniciadores permiten probar casos reales de uso | Si / No |
| El participante realiza al menos un ajuste despues de la primera propuesta | Si / No |

---

## Nota formativa

Este caso de uso es ficticio y se utiliza con fines de aprendizaje. No representa documentacion oficial, procedimientos internos ni herramientas aprobadas por Solclef.
