# Mapa legible del formato de requerimiento vs dato estructurado actual

## Formato y dato estructurado

| Pregunta del formato | Campo(s) estructurado(s) actual(es) |
|---|---|
| Cual es el nombre del proyecto | `nombre_proyecto` |
| Quien es el cliente | `nombre_cliente` |
| Cual es la fecha de levantamiento | `fecha_reunion_inicial` |
| Quien es el responsable del levantamiento | `responsable_comercial` |
| El responsable es decisor | `decisor` |
| A que area o departamento pertenece el proyecto | `area_solicitante` |
| El cliente cuenta con asesor en Microsoft | `asesor_microsoft` |
| Contacto principal del cliente | `contacto_principal_nombre`, `contacto_principal_cargo`, `contacto_principal_correo`, `contacto_principal_tel` |
| Que competencia o soluciones utiliza actualmente el cliente | `competencia_actual` |
| Por que busca el proyecto | `motivo_proyecto` |
| Cuales son los indicadores de exito del proyecto | `indicadores_exito` |
| Stakeholders clave e influencia | `stakeholders` |
| Fecha limite para la entrega | `fecha_limite_entrega` |
| Hitos importantes | `hitos_clave` |
| Presupuesto estimado | `presupuesto_estimado` |
| Proposito principal del proyecto | `objetivo_principal` |
| Problema que se busca resolver | `problema_a_resolver` |
| Por que Raona es la opcion elegida | No existe campo explicito. Hoy podria caer en `observaciones_generales` o `observaciones`. |
| Procesos, sistemas o areas involucradas | `procesos_sistemas_areas_involucradas` |
| Entregables esperados | `entregables_esperados` |
| Que se incluye | `incluye` |
| Que se excluye | `excluye` |
| Cuantos usuarios utilizaran la solucion | `usuarios_estimados` |
| Funcionalidades del sistema | `funcionalidades_clave` |
| Acciones de los usuarios | `acciones_usuario` |
| Reglas de negocio | `reglas_negocio` |
| Quienes usaran la solucion | `usuarios_solucion`, `roles_solucion` |
| Tipo de licenciamiento | `licenciamiento_actual` |
| Terceros o proveedores a integrar | `terceros_integrados` |
| Dispositivos | `dispositivos` |
| Lineamientos de diseno o branding | `lineamientos_branding` |
| Experiencia de usuario esperada | `experiencia_esperada` |
| Fuentes de datos | `fuentes_datos` |
| Integraciones con otros sistemas | `integraciones` |
| Reportes | `reportes` |
| Migracion de datos | `migracion_datos` |
| Riesgos identificados | `riesgos_identificados` |
| Restricciones tecnicas, legales u operativas | `restricciones`, `implicaciones_legales` |
| Medidas de mitigacion | `mitigaciones` |
| Implicaciones legales o de cumplimiento | `implicaciones_legales` |
| Plan y frecuencia de seguimiento | `plan_seguimiento` |
| Observaciones adicionales | `observaciones` |
