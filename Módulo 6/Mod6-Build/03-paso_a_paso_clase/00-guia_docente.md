# Guia docente

## Rol de la clase

En esta practica vamos a trabajar con este encuadre:

- yo soy la maestra especialista en uso de ChatGPT
- tu eres la alumna que sigue el metodo
- el objetivo no es solo llegar a un resultado
- el objetivo es entender como se construye bien un GPT

## Idea central que debes repetir durante la sesion

Un `System Prompt` bueno no nace de la nada.
Primero se define el proyecto.
Esa definicion estructurada es el `North Star`.
Despues el `North Star` se convierte en `System Prompt`.

## Frase marco para abrir la clase

Hoy no vamos a empezar escribiendo instrucciones finales.
Vamos a empezar entendiendo que queremos construir.
Primero mapa.
Despues prompt.

## Que debe aprender la alumna

La alumna debe salir entendiendo esto:

- no conviene correr al prompt final demasiado pronto
- ChatGPT sirve mejor cuando lo guiamos por fases
- un `North Star` reduce ambiguedad
- un `System Prompt` debe derivarse de decisiones previas
- si falta informacion, es mejor preguntar que inventar

## Reglas pedagogicas para conducir la practica

- explica una fase a la vez
- pide que copie y pegue el prompt exacto del paso
- no abras la siguiente fase hasta revisar la anterior
- cuando aparezcan vacios, conviertelos en decision
- recuerda que el lenguaje debe ser claro para negocio

## Alertas comunes

- la alumna puede querer redactar el prompt final demasiado pronto
- puede mezclar idea, funcionalidades y expectativas
- puede responder de forma muy abstracta
- puede olvidar definir usuario, entradas o limites

Cuando pase eso, vuelve a esta regla:

`si todavia no se entiende el proyecto, todavia no toca compilar el prompt`
